# To-Do Backend

Минималистичный бэкенд для приложения списка дел (To-Do App), написанный на чистом Node.js без использования Express.

## Технологический стек

- **Runtime:** Node.js (ES Modules)
- **Database:** SQLite (библиотека `better-sqlite3`)
- **Security:** Хэширование паролей (`crypto.scryptSync` + соль)

## Архитектура проекта

Проект построен по многослойной архитектуре для четкого разделения обязанностей:

- `routes/` — маршрутизация, разбор URL и передача управления сервисам.
- `services/` — бизнес-логика, валидация входных данных, обработка HTTP-статусов (200, 400, 500).
- `models/` — прямой доступ к данным, выполнение SQL-запросов к базе.
- `index.js` — точка входа, запуск HTTP-сервера, настройка CORS и парсинг тела запросов.

## Архитектура API

### Авторизация (Auth)

- `POST /register` — Регистрация нового пользователя.
- `POST /login` — Аутентификация и получение `userId`.

### Списки (Lists)

- `POST /lists` — Создание нового списка.
- `GET /lists?userId=ID` — Получение всех списков конкретного пользователя.
- `PUT /lists/:id` — Редактирование названия и описания списка.
- `DELETE /lists/:id` — Удаление списка.

### Задачи (Todos)

- `POST /todos` — Создание задачи внутри списка.
- `GET /todos?userId=ID` — Получение всех задач пользователя.
- `PUT /todos/:id` — Изменение текста или переключение статуса выполнения задачи (`check`).
- `DELETE /todos/:id` — Удаление задачи.

## Запуск проекта

1. Установите зависимости проекта:

```bash
npm install
```

2. Запустите сервер в режиме разработки:

```bash
npm run dev
```

Сервер запустится на порту `5000`.
